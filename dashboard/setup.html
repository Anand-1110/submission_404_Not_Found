Setup Guide — Real Mode

Follow these 6 steps in order. Each step takes 2–10 minutes. No coding needed!

📧 Email — 5 min 📁 Google Drive — 10 min 📝 Notion — 5 min 📊 Airtable — 5 min 🧪 Test — 1 min 🚀 Run — 1 min
📧 Step 1 of 6 — Email
Set up Gmail to send emails

We use Gmail's App Password — it's like a special password just for this app. Your real Gmail password stays private and safe.

1
Enable 2-Step Verification on Gmail
Open this link: myaccount.google.com/security
Scroll down to "How you sign in to Google" → click "2-Step Verification" → turn it ON.
(If it's already ON, skip this step ✓)
2
Create an App Password
Open this link: myaccount.google.com/apppasswords
In the box, type Scrollhouse → click Create.
Google shows you a 16-character password like: abcd efgh ijkl mnop
Copy it — you'll only see it once!
💡 This is NOT your real Gmail password. It's a special one-time password just for this app. Very safe to use.
3
Paste into your .env file
Open e:\Agent\.env in Notepad or VS Code and fill in these 4 lines:
# Change this to true for Gmail
USE_GMAIL=true

# Your Gmail address
GMAIL_ADDRESS=your.email@gmail.com

# The 16-char App Password you just copied (spaces don't matter)
GMAIL_APP_PASSWORD=abcd efgh ijkl mnop

# Where the manager summary email goes (can be same as above)
MANAGER_EMAIL=your.email@gmail.com

# Also set FROM_EMAIL to your Gmail
FROM_EMAIL=your.email@gmail.com
✅Email step done! Move to Step 2.
📁 Step 2 of 6 — Google Drive
Create a Google Drive Service Account

A "service account" is like a robot Google account. We give it permission to create folders in your Drive automatically.

1
Go to Google Cloud Console
Open: console.cloud.google.com
Sign in with any Google account → Click "New Project"
Give it a name like Scrollhouse → Click Create
2
Enable the Google Drive API
In the top search bar, search for "Google Drive API"
Click on it → Click the blue "Enable" button
Wait a few seconds for it to activate.
📝You might need to go back to your project after enabling. Just click "← Back" or select your project from the top dropdown.
3
Create a Service Account
In the left sidebar → click IAM & Admin → Service Accounts
Click + Create Service Account at the top
→ Name it scrollhouse-agent → Click Done
(You can skip the optional steps 2 and 3 in the form)
4
Download the JSON key file
Click on the service account you just created
→ Click the Keys tab → Add Key → Create new key
→ Choose JSON → Click Create
A file will download automatically (something like scrollhouse-xxxx.json)
Rename that file to credentials.json and move it to e:\Agent\credentials.json
⚠️Keep this file private! Don't share it or commit it to GitHub. It's like a password for the robot account.
5
Copy the service account email address
On the service account page, you'll see an email that looks like:
scrollhouse-agent@scrollhouse-xxxxx.iam.gserviceaccount.com
Copy this email — you'll need it in the next step.
6
Create a Google Drive folder and share it
Open drive.google.com → Click + New → Folder
Name it "Scrollhouse Clients"
Right-click the folder → Share
Paste that service account email → change role to Editor → Click Send
7
Copy the Folder ID into .env
Open the folder in Drive → look at the URL in your browser:
https://drive.google.com/drive/folders/THIS_IS_YOUR_FOLDER_ID
Copy the highlighted part → paste in e:\Agent\.env:
DRIVE_PARENT_FOLDER_ID=paste_the_id_from_the_url_here
✅Google Drive step done! Move to Step 3.
📝 Step 3 of 6 — Notion
Connect Notion to auto-create pages

We create a Notion "Integration" — basically a bot user that can write pages inside your workspace.

1
Create a Notion Integration
Go to: notion.so/my-integrations
Click "+ New integration"
→ Name: Scrollhouse Agent
→ Select your Workspace from the dropdown
→ Click Submit
2
Copy the Integration Token
After creating, you'll see a page with "Internal Integration Secret"
It starts with secret_ — click Show then Copy
Paste it in your e:\Agent\.env file:
NOTION_API_KEY=secret_paste_your_token_here
3
Create a "Clients" page in Notion
In Notion, create a new page and name it "Clients"
This is where all client pages will be created automatically.
You can put this page anywhere in your workspace.
4
Connect your Integration to that page
Open the Clients page → click the ··· (three dots) at the top right
→ Click Connections → Connect to
→ Search for Scrollhouse Agent → click it
→ Confirm by clicking Confirm
📝If you don't do this step, the bot won't have permission to create pages and you'll get a "Not authorized" error.
5
Copy the Page ID into .env
The page ID is in the URL when you open the page. It looks like:
https://notion.so/Clients-abc123def456abc123def456abc12345
Copy the last part after the final dash (32 characters, mix of letters and numbers)
Paste it into e:\Agent\.env:
NOTION_PARENT_PAGE_ID=abc123def456abc123def456abc12345
✅Notion step done! Move to Step 4.
📊 Step 4 of 6 — Airtable
Set up Airtable as your CRM

Airtable is like a smart spreadsheet. We'll create a "Clients" table where every new client gets logged automatically.

1
Create a free Airtable account
If you don't have one, sign up free at airtable.com
Then create a new Base (click "Add a base" → "Start from scratch")
Name it "Scrollhouse CRM"
2
Create the "Clients" table with the right columns
Rename the default table to Clients
Then add these columns (click the + at the end of the column row to add):
Clients table columns — add in this order:
Column Name Field Type
Client NameSingle line text (already exists as "Name")
EmailEmail
Service PlanSingle select → add: Starter, Growth, Enterprise
Start DateDate
Account ManagerSingle line text
StatusSingle select → add: Onboarding, Active, Churned
Drive FolderURL
Notion PageURL
NotesLong text
Onboarded AtDate (turn on "Include time")
SourceSingle line text
3
Get a Personal Access Token
Go to: airtable.com/create/tokens
Click "Create new token"
→ Name: Scrollhouse
→ Add Scopes: click "Add a scope" and add these 3 one by one:
  • data.records:read
  • data.records:write
  • schema.bases:read
→ Add Access: click "Add a base" → select Scrollhouse CRM
→ Click Create token → Copy the token (starts with pat)
AIRTABLE_API_KEY=patXXXXXXXXXXXXXX
4
Copy the Base ID from the URL
Open your Airtable base → look at the URL:
https://airtable.com/appXXXXXXXXXXXXXX/tblXXX...
Copy the appXXXXXXXXXXXXXX part → paste into .env:
AIRTABLE_BASE_ID=appXXXXXXXXXXXXXX
AIRTABLE_TABLE_NAME=Clients
✅Airtable step done! Now test everything.
🧪 Step 5 of 6 — Test
Check all your API keys work

Run this one command — it checks all 4 connections automatically and tells you exactly what's working and what isn't.

1
Open a new PowerShell / Terminal window
Press Win + R → type powershell → press Enter
(Or press Win + X → click Terminal)
2
Run the credential test
POWERSHELL
cd e:\Agent
python agent/test_credentials.py
3
Read the results
You'll see one of these for each service:

✅ Google Drive connected → that service is working
❌ NOTION_API_KEY not set → go back and fill in that key

Fix any ❌ ones by re-checking the step above, then run the test again.
When all 4 show ✅, you're ready!
💡The most common mistake is forgetting to share the Notion page or Drive folder with the integration/service account. Double check those sharing steps!
🚀 Step 6 of 6 — Start
Start the agent and test it for real!

Two commands in two terminal windows, then use the dashboard to fire a real onboarding.

1
Terminal 1 — Start the backend agent
Keep this running. It's the engine that calls all the real APIs.
TERMINAL 1 — keep open
cd e:\Agent
uvicorn agent.main:app --reload --port 8000
You should see: Uvicorn running on http://127.0.0.1:8000
2
Open the Dashboard
The dashboard server is already running at:
→ http://localhost:7890
📝If it's not running, open Terminal 2 and run: python -m http.server 7890 --directory e:\Agent\dashboard
3
Fill the form and fire a REAL onboarding!
In the dashboard: fill in the New Client Intake form with real details
→ Use your own email as the client email as a test
→ Click "Trigger Onboarding Agent"

Within 10–15 seconds you will see:
📧 A real welcome email arrive in your inbox
📁 A real folder appear in your Google Drive
📝 A real page created in your Notion workspace
📊 A real row added to your Airtable Clients table
✉️ A completion summary email arrive at your MANAGER_EMAIL
🎉 That's it! Your onboarding agent is fully live. Every new client form submission now triggers the entire onboarding flow automatically — no manual work needed.

🎯 You're fully set up!

The entire Scrollhouse client onboarding process now runs automatically.
